# Workflow: Створення нової секції теми

Цей воркфлоу описує правильний порядок дій для додавання нових компонентів (секцій) до нашої WordPress теми. 
Згідно з архітектурою проєкту ми спочатку розробляємо компонент у середовищі Gulp (статичний HTML/SCSS), і тільки після затвердження дизайну переносимо його у PHP-шаблони та підключаємо ACF.

## Крок 1: Створення статичного прототипу (Gulp)
1. Створіть файл розмітки секції в `assets/html-dev/sections/назва-секції.html`.
2. Заповніть HTML необхідними BEM-класами.
3. Використайте placeholder-тексти та заглушки (наприклад, для картинок).
4. Підключіть цю секцію у потрібному шаблоні, наприклад в `assets/html-dev/index.html`: `@@include('sections/назва-секції.html')`.

## Крок 2: Написання стилів
1. Створіть SCSS-файл у `assets/scss/sections/_назва-секції.scss`.
2. Додайте імпорт цього файлу до `assets/scss/main.scss`: `@use 'sections/назва-секції';`.
3. Напишіть усі необхідні стилі для компонента, використовуючи методологію BEM.
4. Дочекайтеся перевірки користувачем змін на локальному сервері (`gulp`).

## Крок 3: Інтеграція в WordPress (PHP + ACF)
**Важливо: Виконуйте цей крок лише після затвердження статичної версії!**
1. Скопіюйте затверджену HTML-розмітку у новий файл `sections/назва-секції.php`.
2. Замініть жорстко задані (placeholder) тексти та медіа-дані на динамічний вивід з полів ACF (наприклад, використовуючи масив `$args`, якщо він туди передається через `get_template_part()` у `front-page.php`, або через `get_field()`).
3. **Обов'язкова безпека:** Використовуйте функції екранування (`esc_html`, `esc_url`, `wp_kses_post`) для всіх змінних перед їхнім виводом.

## Крок 4: Завершення
- Надішліть результати користувачу для перевірки динамічних даних в адмінці WordPress.
